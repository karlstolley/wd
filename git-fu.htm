Git Fu: Basic Git Commands

This page will be added to as we get more sophisticated with Git in class.

Creating a Git Repository

Change directories into the directory that contains all of your project files, and run git init: 

$ cd htdocs
$ git init
Initialized empty repository in /home/username/htdocs

Checking on the Status of the Repository

$ git status

Adding a File or Files to Commit

Add files with the git add command, followed by either a dot, . (all files) or a space-separated list of files:

$ git add .
$ git add index.htm screen.css

Committing Added Files

Once the files that are to be committed have been added to the index, run git commit -m followed by a short message to record with the commit:

$ git commit -m "First files for my website"

Checking the Log of Commits

$ git log

Adding A Remote

Note: This is not necessary if you have cloned a repository. To see if you have any remotes, run git remote by itself. To check the address of the remote, run cat .git/config. Be sure to replace username and repositoryname to reflect your username and repository:

$ git remote add origin git@github.com:username/repositoryname.git

Pulling from a Remote

Note: Do this each time you sit down to work at your computer, if you’ve been working elsewhere. Replace master with the name of the branch you’re working on, if it’s different from master:

$ git pull origin master

Pushing to a Remote

Note: Do this before you log out of the computer you’re working on, after you’ve committed all of your work.

$ git push origin master
